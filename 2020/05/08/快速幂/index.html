快速幂

May 8, 2020 #study

做题的时候后遇到了快速幂问题，代码不长，属于看了就会，不看就不会的问题。

问题描述

编写函数，求解pow(x,n)

问题分析

首先第一反应是暴力解法，不断相乘直到得到结果，但是这样做的时间复杂度是O(n)，在一些大数时难以接受。接下来首先介绍快速幂递归的方法：

以7的十次幂为例：假如我们正常的理解下,不就是7乘个十次嘛，这样当然可以，也足以应对很多场景了。但是当a^n中的n非常大时，这个O(n)的幂运算好像不太够，那有没有什么办法能够降低幂运算的时间复杂度呢？

第一步：先算7的5次方，即7∗7∗7∗7∗7，再算它的平方，共进行了55次乘法。但这并不是最优解，因为对于“7的5次方”，我们仍然可以拆分问题。

第二步：先算7∗7得4949，则7的5次方为49*49*749∗49∗7，再算它的平方，共进行了4次乘法。模仿这样的过程，我们得到一个在 时间内计算出幂的算法，也就是快速幂。(此处借用了部分知乎上的内容)  方法二运用的无非是一个二分的思路，因此我们可以很轻松的得到一个递归方程:

  因此我们可以比较自然的写出递归解法如下：

using LL = long long;
using db = double;
class Solution {
    db qpow(db a, LL n){
        if (n < 0) 
            return qpow(1/a, -n);
        if (!n) 
            return 1.0;
        else if (n & 1) //判断奇偶数
            return qpow(a, n - 1) * a;
        else {
            double tmp = qpow(a, n / 2);
            return tmp * tmp; //假如直接return qpow(a, 	n/2) * qpow(a, n/2)，那还是计算了两次，对于复杂度就没有降低了~
        }
    }
public:
    double myPow(double x, int n) {
        return qpow(x, n);
    }

迭代优化

递归算法还有额外的空间开销，所以我们考虑是否可以用迭代的方式解决问题。观察下图，我们可以发现，可以利用二进制数来判断，当前位置对最终结果是否有贡献：

using LL = long long;
class Solution {
public:
    double myPow(double x, int n) {
        LL t;
        double ans(1.0); // initialize
        if (n < 0) 
            x = 1/x;
        t = LL(std::abs(n));
        while (t){
            if (t & 1) 
                ans *= x; // 假如低位为1,则乘上x
            x *= x; // x自乘
            t = t >> 1; // 右移一位
        }

        return ans;
    }
};

注意事项

由于是要解决大数的求次幂问题，int型显然不足以。另外，底数也可能是小数，int型同样不合适。所以我们采用long long型来存储。同时要注意某些情况的特判。比如0次幂等。
