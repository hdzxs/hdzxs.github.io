PAT_A_1055

Forbes magazine publishes every year its list of billionaires based on the annual ranking of the world’s wealthiest people. Now you are supposed to simulate this job, but concentrate only on the people in a certain range of ages. That is, given the net worths of N people, you must find the M richest people in a given range of their ages.

Input Specification:

Each input file contains one test case. For each case, the first line contains 2 positive integers: N (≤105) - the total number of people, and K (≤103) - the number of queries. Then N lines follow, each contains the name (string of no more than 8 characters without space), age (integer in (0, 200]), and the net worth (integer in [−106,106]) of a person. Finally there are K lines of queries, each contains three positive integers: M (≤100) - the maximum number of outputs, and [Amin, Amax] which are the range of ages. All the numbers in a line are separated by a space.

Output Specification:

For each query, first print in a line Case #X: where X is the query number starting from 1. Then output the M richest people with their ages in the range [Amin, Amax]. Each person’s information occupies a line, in the format

Name Age Net_Worth

The outputs must be in non-increasing order of the net worths. In case there are equal worths, it must be in non-decreasing order of the ages. If both worths and ages are the same, then the output must be in non-decreasing alphabetical order of the names. It is guaranteed that there is no two persons share all the same of the three pieces of information. In case no one is found, output None.

Sample Input:

12 4
Zoe_Bill 35 2333
Bob_Volk 24 5888
Anny_Cin 95 999999
Williams 30 -22
Cindy 76 76000
Alice 18 88888
Joe_Mike 32 3222
Michael 5 300000
Rosemary 40 5888
Dobby 24 5888
Billy 24 5888
Nobody 5 0
4 15 45
4 30 35
4 5 95
1 45 50

Sample Output:

Case #1:
Alice 18 88888
Billy 24 5888
Bob_Volk 24 5888
Dobby 24 5888
Case #2:
Joe_Mike 32 3222
Zoe_Bill 35 2333
Williams 30 -22
Case #3:
Anny_Cin 95 999999
Michael 5 300000
Alice 18 88888
Cindy 76 76000
Case #4:
None

问题解析

依旧是一个排序问题，但是观察这题会发现，他输入的数字和最多要显示的数字差距很大，10的5次幂和100，所以可以在输入之后进行一次筛选，只留下每个年龄的前一百名，因为100名之后，无论如何不会被输出。（实际上，这题不这么处理就会超时，我醉了）

注意事项

  1. 排序的时候，即按照要求的排序方式排序，最后寻找符合条件的输出即可。
  2. 需要两个数组，users[]和valid[]，分别装筛选前后的用户。
  3. 需要一个age数组，用来存储每一个年龄的人数。

代码实现

#include<iostream>
#include<string>
#include<algorithm>
using namespace std;

int age[200] = { 0 };

struct User {
	string name;
	int age;
	int worth;
}users[100000],valid[100000];    //第一个表示全部人员，第二个表示有效人员，每一个年龄最多有效100个

bool cmp(User a, User b) {
	if (a.worth != b.worth)
		return a.worth > b.worth;
	else if (a.age != b.age)
		return a.age< b.age;
	else
		return  a.name.compare(b.name) < 0;
}

int main() {
	int n, k;
	int maximun, Amin, Amax;
	cin >> n >> k;

	for (int i = 0; i < n; i++) {
		cin >> users[i].name >> users[i].age >> users[i].worth;
	}

	sort(users, users + n, cmp);          //排序

	//对于每种年龄的存入有效数组，最多100个

	int validNum = 0;

	for (int i = 0; i < n; i++) {        //统计每个年龄的数量，把有效的存入valid数组
		if (age[users[i].age] < 100) {     //某个年龄的数目少于100
			age[users[i].age]++;           //数字加一
			valid[validNum++] = users[i];  //总有效人数增加，同时将有效人录入
		}
	}

	for (int i = 0; i < k; i++) {
		cin >> maximun >> Amin >> Amax;

		cout << "Case #" << i + 1<< ":" << endl;
		int printNum = 0; 
		for (int j = 0; printNum < maximun && j < validNum; j++) {
			if (valid[j].age >= Amin && valid[j].age <= Amax) {
				printNum++;
				cout << valid[j].name<< " "  << valid[j].age << " " << valid[j].worth << endl;
			}
		}

		if (printNum == 0)
			cout << "None" << endl;
	}


	return 0;
}
