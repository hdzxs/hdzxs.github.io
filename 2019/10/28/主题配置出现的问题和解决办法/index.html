casper主题配置出现的问题和解决办法

Oct 28, 2019 #study #hugo

  上一周花了好久终于弄懂了hugo中的主题是怎么用的，配置过程不算复杂，但初次尝试还是遇到了许多问题，这里做个记录。

1. 配置文件

  Hugo中的主题的下载通过git即可，在下载成功之后，针对自己的博客还要进行具体的配置。首先，把主题的配置文件config.toml复制到博客根目录下，然后参照主题给出的Readme文档修改各个参数的内容。（有的参数是hugo内置的，可以通过参考官方文档学习）

BaseUrl= "https://hdzxs.github.io/"
LanguageCode= "fr-FR"
Title= "Cooper's Blog"
paginate = 5
pluralizelisttitles = false
theme = "casper"
Copyright = "All rights reserved - 2019"
canonifyurls = true
hasCJKLanguage = true
preserveTaxonomyNames = true

[params]
  description = "一个让自己说些废话，悄悄学习的小地方"
  cover = "images/cover1.jpg"
  author = "Cooper"
  authorlocation = "Shenyang Liaoning"
  authorwebsite = "https://hdzxs.github.io/"
  bio = "Student in Computer Sicence，OUC"
  authorbio = "Student in Computer Sicence，OUC"
  logo = "images/logo4.png"
  githubName = "hdzxs"
  email="1148665369@qq.com"
  hideHUGOSupport = false
  
    //主页菜单内容
[[menu.main]]
  name = "Blog"
  pre = "<h3>Blog</h3>"
  weight = -1
  identifier = "index"
  url = "/"
    
    //永久连接格式
[permalinks]
  post = "/:year/:month/:day/:filename/"

2.大小写问题

  Hugo会默认吧URL和categgories中的大写转为小写，可以通过下面两个配置取消：

disablePathToLower = true
preserveTaxonomyNames = true

3.posts显示问题

  当时问题是这样的，这个主题的首页正常应该显示博文的列表，但是我这里显示了Posts这个类的名字，必须点击进去才能看到文章，我试图过通过修改JS文件解决，奈何根本看不懂；国内的Hugo关键词搜出来的又都是最基本的建站教程，根本没有这个问题。最后在论坛上找到了这个问题的结局办法。（以后还是要多看这些外国网站，比国内的信息全多了，而且这个水平的英语阅读起来问题不大）

  这是主题版本老旧引起的问题，将themes\casper\layouts\partials\home-content.html中的 .Data.Pages替换为.Site.RegularPages就解决了。

    {{ $list := where .Site.RegularPages "Section" "in" ($.Scratch.Get "paginatedSections") }}
    {{ $list := where $list "Section" "!=" "" }}
    {{ $paginator := .Paginate ( $list ) }}

  原答案出处： https://discourse.gohugo.io/t/hugo-casper-two-theme-home-page-redirect-issue/20817

4.上传到github

  上传到github通过的是git命令，可以将其写成一个脚本，这样以后只要运行这个文件就可以直接将博客push到github上了。

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# build the project
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

cd public

git add .

msg="rebuilding site `date`"

if [ $# -eq 1 ]
  then msg="$1"
fi

git commit -m "$msg"

# push source to github

git push upstream master

# come back to blog root

cd ..

  赋予deploy.sh文件执行权限，之后执行./deploy.sh就可以了。

5.图片路径问题

解决这个问题，主要是两个方面：

第一、在config.toml中配置baseurl。作用是无论在本地还是发布出去，图片的路径都是相对于baseurl而言的。

第二、在Markdown图片路径中，以static为根目录写全路径。

示例：假设测试baseurl为http://localhost:1313/，图片位置：/static/img/hugo/123.png，那么在Markdown中写作![图片说明](/img/hugo/123.png)。这时，在浏览器就可以看到图片了 。
