蓝桥杯练习——小计算器

Oct 30, 2019 #study

  昨天花了一个晚上做了小计算器这个练习题，中间出现了好多问题，在这里记录下来。

题目重述

问题描述

　　模拟程序型计算器，依次输入指令，可能包含的指令有

　　1. 数字：’NUM X’，X为一个只包含大写字母和数字的字符串，表示一个当前进制的数 　　2. 运算指令：’ADD’,‘SUB’,‘MUL’,‘DIV’,‘MOD’，分别表示加减乘，除法取商，除法取余 　　3. 进制转换指令：’CHANGE K’，将当前进制转换为K进制(2≤K≤36) 　　4. 输出指令：’EQUAL’，以当前进制输出结果 　　5. 重置指令：’CLEAR’，清除当前数字

　　指令按照以下规则给出： 　　数字，运算指令不会连续给出，进制转换指令，输出指令，重置指令有可能连续给出 　　运算指令后出现的第一个数字，表示参与运算的数字。且在该运算指令和该数字中间不会出现运算指令和输出指令 　　重置指令后出现的第一个数字，表示基础值。且在重置指令和第一个数字中间不会出现运算指令和输出指令 　　进制转换指令可能出现在任何地方

　　运算过程中中间变量均为非负整数，且小于2^63。 　　以大写的’A’~‘Z’表示10~35

输入格式

　　第1行：1个n，表示指令数量 　　第2..n+1行：每行给出一条指令。指令序列一定以’CLEAR’作为开始，并且满足指令规则

输出格式

　　依次给出每一次’EQUAL’得到的结果

样例输入

7 CLEAR NUM 1024 CHANGE 2 ADD NUM 100000 CHANGE 8 EQUAL

样例输出

2040

解题思路

  最开始一下想到了之前讲过的计算器实现：利用队列的方式，但是，后来发现不需要这么麻烦，只是一个字符串匹配，然后执行具体操作的过程，重点应该在于十进制和各种进制的转换。

解题代码

#include<iostream>
#include<string>
#include<vector>
#include<math.h>
using namespace std;


long long  numtoten(int k) {				//num函数将输入的k进制的number转为十进制
	string ss;
	cin >> ss;
	long long num = 0,kk = 1;

	for (int i = 0; i < ss.length(); i++) {

		if (ss[ss.length() - i - 1] > '9') {
			num += (ss[ss.length() - i - 1] - 'A' + 10)* kk;
		}
		else{
			num += (ss[ss.length() - i - 1] - '0') * kk;
		}
		kk *= k;
	}
	return num;
	
}

void output(long long num, int k) {		//将结果以当前进制输出
	if (num == 0)
		cout << '0';
	else {
		char ans[66] = { 0 };
		int i = 0;
		char numI;
		while (num >= k) {
			if (num%k < 10) {
				numI = num % k + '0';
				num /= k;
			}
			else if (num%k >= 10) {
				numI = num % k - 10 + 'A';
				num /= k;
			}
			ans[i] = numI;
			i++;
		}

		if (num < 10)
			ans[i] = num + '0';
		else if (num >= 10)
			ans[i] = num - 10 + 'A';


		for (int j = i; j >= 0; j--) {
			cout << ans[j];
		}
	}
	cout << endl;
}

int main() {
	int index = 0;
	int op = 0;
	long long num = 0;        //num为结果
	int k = 10;			//k为当前进制,初始为10进制
	string cal;
	cin >> index;
	while (index--) {
		cin >> cal;
		if (cal == "CLEAR")
			op = num = 0;
		else if (cal == "ADD")
			op = 1;
		else if (cal == "SUB")
			op = 2;
		else if (cal == "MUL")
			op = 3;
		else if (cal == "DIV")
			op = 4;
		else if (cal == "MOD")
			op = 5;
		else if (cal == "CHANGE")
			cin >> k;
		else if (cal == "NUM") {
			switch (op) {
			case 0:num = numtoten(k); break;
			case 1:num += numtoten(k); break;
			case 2:num -= numtoten(k); break;
			case 3:num *= numtoten(k); break;
			case 4:num /= numtoten(k); break;
			case 5:num %= numtoten(k); break;
			}
		}
		else if (cal == "EQUAL")
			output(num, k);
	}
	return 0;
}

出现问题

  第一个最尴尬的问题，蓝桥杯的练习系统给的输入输出是正确的输入输出，我一直以为是我自己的，纠结了半天。

  卡住我的有两个地方：一是数字大小的问题。他给的大小到2^64，明显超过了int的范围，所以用int后面会报错，改用了longlong型数据，感觉这些算法题总有这种情况。第二个是进制转换，我想到了要从后往前转换，但是判断那里居然写反了，看了好久没看出来，太菜了。

进制转换

  最后在这里记下一个比较标准的进制转换的代码实现：

int Atoint(string s,int radix)    //s是给定的radix进制字符串
{
    int ans=0;
    for(int i=0;i<s.size();i++)
    {
        char t=s[i];
        if(t>='0'&&t<='9') ans=ans*radix+t-'0';
        else ans=ans*radix+t-'a'+10;
    }
        return ans;
}
